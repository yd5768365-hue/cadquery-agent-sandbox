​这是一个基于 Cherry Studio 构建的高度工程化的 AI Agent 系统。它专门用于 CadQuery 参数化建模，通过 MCP (Model Context Protocol) 协议实现了代码编写、沙盒运行、文件覆盖的一体化流程，并具备严格的安全隔离机制。
​🌟 项目核心亮点
​闭环自进化：AI 不仅仅是写代码，它能通过 /bin/bash 在本地容器中执行代码，读取报错，并自动修复。
​沙盒隔离 (Sandbox)：所有代码运行均在 Docker 容器内进行，防止潜在的恶意代码对宿主机系统造成损害。
​权威知识对齐：通过注入 CadQuery 官方文档，极大地降低了 LLM 产生“幻觉”的概率，确保生成的 API 调用准确无误。
​工程化提示词 (System Prompt)：内置了一套包含工程哲学、作业程序（SOP）以及权限反馈机制的顶级 Prompt
​🏗️ 系统架构
​项目的运行逻辑如下：
​用户输入：在 Cherry Studio 提出建模需求。
​大脑思考：DeepSeek 模型结合 CadQuery 权威文档进行逻辑推理。
​工具调用 (MCP)：
​read_file: 读取项目上下文。
​write_file: 覆盖或新建建模脚本。
​run_code: 在 Docker 容器内执行 Python 脚本。
​bash_access: 必要的系统级环境准备（受限权限）。
​反馈闭环：捕获容器输出的日志，AI 根据运行结果决定是交付作品还是进入下一轮 Bug 修复。

📜 交付风格与工程哲学
​本项目不仅是一个工具，更是一套建模准则。它追求：
​模块化：生成的 CAD 代码必须易于后续修改。
​确定性：所有的操作必须有明确的反馈反馈机制。
​前置追诉：严禁在未确认权限的情况下尝试越权操作。
